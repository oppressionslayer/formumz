What is this?

A live GF(2) constraint system over an N×N lattice of hidden bits. Trusted laws under-determine the state on purpose; witnesses fill in the rest — but some lie. The system runs real syndrome decoding (loopy belief propagation + min-weight enumeration) to identify the liars, then bootstrap-verifies the recovered state against held-out trusted laws.

Every number in the HUD comes from actual GF(2) elimination running in your browser — no mock data.

Quick start
  1. Click a Scenarios button — Easy places t*−1 lies (under the identifiability bound), Tight places exactly t* (uniquely decodable), Ambiguous places t*+1 (multiple candidates), Overwhelmed places t*+2 (graceful failure).
  2. Watch the HUD's detected k/m badge. ✓ means the decoder caught every actual liar.
  3. Drag the Phase scrubber to animate the two-phase collapse: shimmer (superposition) → intra-block forced → cross-block forced → final.
  4. For ambiguous scenarios, turn on Active interrogation: amber rings mark cells whose true value will maximally distinguish the candidate liar sets. Click one to inspect.
  5. space re-seeds the world. Click any cell to probe (measure or flip).
What the colors mean
  • cyan edges = trusted laws (T_A) used for detection
  • faded edges = trusted laws held out for bootstrap (T_B)
  • violet edges = witnesses (the suspect pool)
  • pulsing red edges = corrupted by the liar coalition
  • iridescent shimmer cell = still in superposition (free variable)
  • 0 / 1 glyph cell = collapsed (forced by algebra)
  • cyan ring around cell = detector flagged a variable in a found liar
  • amber ring = active-interrogation suggestion
Why it's useful

A sandbox for the forensic syndrome-decoder pipeline from lars-logic. Useful for: teaching how identifiability bounds work (d, t*); demonstrating that naive residual heuristics fail on adversarial coalitions; testing scenarios where a knowledge base's witnesses are partially corrupted; understanding the relationship between code minimum distance and how many lies you can catch.

Identifiability theorem

For the projected syndrome code with minimum distance d:

t* = ⌊(d − 1) / 2⌋

Up to t* adversarial witnesses can be uniquely identified from the observable parity syndromes. At weight t*+1, the syndrome admits multiple minimum-weight explanations — the system can return at best the equivalence class. Beyond that, identifiability is lost.
